# Pull Request Checklist

Before requesting a review, confirm each item:

- [ ] My code builds/runs without errors
- [ ] No debugging print statements left in code
- [ ] I tested my code on at least one real input
- [ ] I pulled latest `main` and resolved merge conflicts locally
- [ ] My branch name reflects the task
- [ ] My PR title describes the change clearly
- [ ] I requested at least 1 reviewer (not myself)
- [ ] Anyone using my code is also added as a reviewer
- [ ] I updated documentation if needed
- [ ] I updated the Kanban board task status

**After approval:**
- [ ] I merged using GitHub PR (not terminal push to main)
- [ ] I deleted my branch
- [ ] I updated local main (`git pull`)
